How to use bun (complete linux version) on powershell core

Yesterday I decided to give bun a try as a replacement for npm on a personal side project. However I use Windows 11 as my daily driver and the windows version is not feature complete.

The main option is to use WSL to run bun, but I wanted to continue to use powershell as the only terminal I use on windows, so after some search and messing around I figured this solution:

Part 1. If you don't have it already, enable wsl and install a distro from ms store (I'm using ubuntu)

Part 2. Open your wsl terminal and install bun using the bash script from https://bun.sh/package-manager

Part 3. On your wsl terminal, edit the file ~/.bashrc using any text editor and comment out the lines containing export BUN_INSTALL="$HOME/.bun" and export PATH=$BUN_INSTALL/bin:$PATH by addin a # to the beggining of each line

Part 3. Copy content from bun folder .bun/bin/ to /usr/bin/ using something like:
sudo cp .bun/bin/bun /usr/bin/bun
sudo cp .bun/bin/bunx /usr/bin/bunx

Part 4. On your powershell, install Install-Module WslInterop

Part 5. Open your powershell profile and add Import-WslCommand "bun"

After all theses steps, just re-open your powershell and bun should be available like it was installed on your windows.
